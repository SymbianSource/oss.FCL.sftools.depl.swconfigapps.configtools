Introduction

What is Carbide.v?

Carbide.v is a Standalone tool for Nokia OPMs (later on for Operators and possibly Licencees) for preparing phone software variant proposals (fill out software setting values). Featuring for instance:


  • Instant input data validation
  • Comparison and reuse of setting values
  • Default values and descriptions (= field helps)
  • Preview of pictures, animations, themes, sounds etc.
  • Communication about individual settings

Carbide.v Concepts

Configuration

The settings and all their selectable values available for a software Variant.

cpf

The container of the settings with their values and resources of one software variant as one file.

Resource

An asset such as picture, sound, animation, video or theme included in the variant proposal.

Setting

One customizable element in Software Configuration; a setting encloses both the information on the current value and for instance description (field help), available options, or minimum and maximum values.

Vanilla Variant Proposal

The software configuration in the initial CPF, and its settings that will be used as Default Settings for further customization. It is initially created by a Phone Program and used as a base for for instance country or operator specific software variant proposal by an OPM or Operator.

Variant

The phone, its sales package, memory card, physical phone settings and software image that will be sent to production line. Carbide.v is the tool for creating software variant proposal.

Variant Proposal

The 'order form' that includes software setting values and references to resources. Prepared in Carbide.v and processed as an attachment in MDC.
